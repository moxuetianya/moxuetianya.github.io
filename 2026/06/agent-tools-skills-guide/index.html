环境准备（前置依赖）

本节涉及的工具依赖 npx/npm/node 和 uvx/uv，请先确保以下环境就绪。

Node.js + npm + npx 安装

npx 随 Node.js 自带，安装 Node.js 即可获得。

方法一：NodeSource（推荐，Linux）

# 安装 Node.js 22 LTS
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version    # v22.x
npm --version     # 10.x
npx --version     # 10.x

方法二：nvm（多版本管理）

# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc

# 安装并使用 Node.js 22
nvm install 22
nvm use 22
nvm alias default 22

方法三：fnm（快速版本管理，Rust 实现）

# macOS/Linux
curl -fsSL https://fnm.vercel.app/install | bash
source ~/.bashrc

fnm install 22
fnm default 22
fnm use 22

方法四：直接下载二进制（Linux 通用）

# 下载 Node.js 22 LTS 预编译包
wget https://nodejs.org/dist/v22.19.0/node-v22.19.0-linux-x64.tar.xz
tar -xf node-v22.19.0-linux-x64.tar.xz
sudo mv node-v22.19.0-linux-x64 /usr/local/nodejs
sudo ln -sf /usr/local/nodejs/bin/node /usr/local/bin/node
sudo ln -sf /usr/local/nodejs/bin/npm /usr/local/bin/npm
sudo ln -sf /usr/local/nodejs/bin/npx /usr/local/bin/npx

npm 镜像源（Mirror Registry）配置

国内网络环境下建议配置 npm 镜像加速。以下为常用源：

镜像URL说明
npmmirror（阿里云）https://registry.npmmirror.com国内最全，推荐
腾讯云https://mirrors.cloud.tencent.com/npm/腾讯维护
华为云https://repo.huaweicloud.com/repository/npm/华为维护
npm 官方https://registry.npmjs.org默认（需代理）

方法一：npm config 设置（全局生效）

# 设置镜像（以阿里云 npmmirror 为例）
npm config set registry https://registry.npmmirror.com

# 验证
npm config get registry

# 恢复官方源
npm config set registry https://registry.npmjs.org

方法二：nrm（源管理工具）

npm install -g nrm

nrm ls                          # 列出所有可用源
nrm use npmmirror               # 切换到阿里云镜像
nrm test                        # 测速

方法三：单次使用（不修改全局配置）

npm install --registry=https://registry.npmmirror.com <package>
npx --registry=https://registry.npmmirror.com skills add <repo>

npm 代理配置（如需代理访问外网）

npm config set proxy http://192.168.2.100:10808
npm config set https-proxy http://192.168.2.100:10808

# 取消代理
npm config delete proxy
npm config delete https-proxy

Python + uv/uvx 安装

部分 MCP 工具（如 mcp-atlassian）使用 Python，通过 uvx 运行。

安装 uv（Python 包管理器）：

# Linux/macOS
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc

# 或使用 pip 安装
pip install uv

uv pip 镜像配置：

# 设置 PyPI 镜像（阿里云）
export UV_INDEX_URL=https://mirrors.aliyun.com/pypi/simple/

# 或在 uv 配置文件中 ~/.config/uv/uv.toml
echo '[pip]
index-url = "https://mirrors.aliyun.com/pypi/simple/"' > ~/.config/uv/uv.toml

统一安装工具：npx skills

GitHub: https://github.com/vercel-labs/skills (⭐22k+)

npx skills 是 Vercel 推出的跨代理 Skills 统一安装工具，支持 OpenCode、Claude Code、Codex、Cursor 等 70+ 种代理。

# 基本用法
npx skills add <owner/repo> [-a <agent>] [--skill <name>] [--global]

# 支持的代理名称：opencode, claude-code, codex, cursor, ...

# 从 GitHub 安装
npx skills add JimLiu/baoyu-skills -a opencode

# 指定特定 skill
npx skills add anthropics/skills --skill skill-creator -a claude-code

# 全局安装（用户级别）
npx skills add vercel-labs/agent-skills -a opencode --global

# 列出可用 skills（不安装）
npx skills add garrytan/gstack -a codex --list

简介完整的 AI 编程代理开发方法论与 Skills 集合。包含 TDD、子代理驱动开发、系统调试、Git Worktree 隔离开发、Code Review、计划编写等 25+ Skills，覆盖从需求分析到合并发布的全流程
类型Plugin + Skills（支持 Claude Code / Codex / OpenCode / Cursor / Gemini CLI / GitHub Copilot CLI 等 7+ 代理）

# 在 Codex 中输入 /plugins，搜索 superpowers，选择 Install Plugin

# Cursor

copilot plugin marketplace add obra/superpowers-marketplace
copilot plugin install superpowers@superpowers-marketplace

# Factory Droid
droid plugin marketplace add https://github.com/obra/superpowers
droid plugin install superpowers@superpowers

核心工作流

  1. brainstorming — 编码前激活，Socratic 式需求澄清，分段输出设计文档供确认
  2. using-git-worktrees — 设计确认后创建隔离工作空间，检查测试基线
  3. writing-plans — 将工作拆分为 2-5 分钟的 bite-sized 任务，含精确文件路径和验证步骤
  4. subagent-driven-development — 每个任务派遣独立子代理，两阶段审查（规格合规 + 代码质量）
  5. test-driven-development — 强制执行 RED-GREEN-REFACTOR 循环，先写失败测试再写最小实现
  6. requesting-code-review — 任务间自动触发审查，严重问题阻塞进展
  7. finishing-a-development-branch — 任务完成后验证测试、清理 worktree、提供 merge/PR 选项

代理在执行任何任务前会自动检查并加载相关 Skill，非建议性工作流，而是强制执行。

MCP 类工具

MCP（Model Context Protocol）服务器以独立进程运行，通过标准 I/O 或 HTTP 与代理通信，提供工具调用能力。

Playwright CLI + Skills（编码代理推荐）⭐

项目详情
GitHubhttps://github.com/microsoft/playwright-cli
简介Playwright CLI 是面向编码代理的浏览器自动化入口。相比 MCP，CLI+Skills 模式更省 token，适合需要同时处理大量代码的编码代理
类型CLI + Skills（Node.js/npm）

安装方法

# 全局安装 CLI
npm install -g @playwright/cli@latest

# 验证
playwright-cli --help

安装 Skills

# 为代理安装 Playwright Skills（代理将自动发现本地 Skills）
playwright-cli install --skills

# 或通过 npx skills（OpenCode / Codex）
npx skills add microsoft/playwright-cli -a opencode
npx skills add microsoft/playwright-cli -a codex

基本命令：

playwright-cli open "https://example.com"          # 打开页面
playwright-cli open "https://example.com" --headed  # 有头模式（可见浏览器）
playwright-cli type "input text"                    # 输入文本
playwright-cli press Enter                          # 按键
playwright-cli click "按钮文本"                     # 点击
playwright-cli check e21                            # 勾选复选框
playwright-cli screenshot                           # 截图
playwright-cli snapshot                             # 页面快照
playwright-cli close                                # 关闭浏览器

MCP vs CLI 选择建议：编码代理（Claude Code / OpenCode / Codex）推荐使用 CLI，MCP 更适合需要持久浏览器状态的自愈测试、长时间探索性自动化等场景。

16. SSH MCP — tufantunc/ssh-mcp

项目详情
GitHubhttps://github.com/tufantunc/ssh-mcp
NPMssh-mcp
简介轻量 SSH MCP 服务，支持命令执行和 sudo 提权，可配置超时和命令长度限制
类型MCP Server（Node.js/npx）
# OpenCode（命令行）
opencode mcp add ssh-mcp -- npx -y ssh-mcp -- --host=1.2.3.4 --user=root --password=pass --timeout=30000

# OpenCode（手动编辑 ~/.config/opencode/opencode.json）：
{
  "mcpServers": {
    "ssh-mcp": {
      "command": "npx",
      "args": [
        "ssh-mcp", "-y", "--",
        "--host=1.2.3.4",
        "--port=22",
        "--user=root",
        "--password=pass",
        "--timeout=30000"
      ]
    }
  }
}

# Claude Code
claude mcp add --transport stdio ssh-mcp -- npx -y ssh-mcp -- --host=YOUR_HOST --user=YOUR_USER --password=YOUR_PASSWORD

# Codex
codex mcp add ssh-mcp -- npx -y ssh-mcp -- --host=YOUR_HOST --user=YOUR_USER --password=YOUR_PASSWORD

私钥认证 / sudo 提权

# 私钥认证
--key=/path/to/private/key

# sudo 提权
--sudoPassword=your_sudo_pass

# 禁用 sudo 功能
--disableSudo

# 自定义超时（默认 60000ms）
--timeout=120000

# 取消命令长度限制
--maxChars=none

跳板机/Bastion✅ dual transport❌
命令黑白名单✅❌ (长度限制)
